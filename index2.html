데이터 입력 프로세스 편의 개선

TMS 내에서 발생하는 수동 오더 입력 작업을 간편화하고,
TMS와 실시간으로 연동하여 업무 효율성과 데이터 정확성을 향상시킵니다.

문제점 (As-Is)

  • ✘
    오입력 가능성: 계약명, 고객사 등 텍스트 직접 입력 시 인적 오류(Human Error) 발생 위험
  • ✘
    데이터 단절: Excel 입력 후, TMS 시스템에 별도로 업로드해야 하는 이중 작업 발생
  • ✘
    낮은 편의성: 입력 가능한 항목을 매번 확인하거나, 다른 문서를 참조해야 하는 불편함
  • ✘
    지연된 오류 확인: 필수값이 누락되어도 TMS 시스템에 입력하는 시점에야 오류를 인지 가능

개선 방안 (To-Be)

  • ✔
    동적 드롭다운 메뉴: 셀 선택 값에 따라 다음 선택 목록을 가져와 드롭다운으로 제공
  • ✔
    원클릭 데이터 전송: 상태 값 변경으로 해당 행의 데이터가 즉시 TMS로 자동 전송 및 저장
  • ✔
    실시간 유효성 검증: '확정' 시 필수값 누락 여부를 먼저 체크하고, 사용자에게 즉시 피드백
  • ✔
    처리 결과 시각화: 전송 성공/실패 여부를 명확하게 표시하여 직관성 향상

onEdit(e) Simple Trigger

사용자가 셀을 수정할 때마다 자동으로 실행되는 이벤트 핸들러입니다. 별도 설정 없이 즉시 동작하며, 이벤트 객체 `e`를 통해 변경된 셀의 위치, 값 등의 정보를 받아 로직을 분기합니다.

function onEditCell(e) {
  var range = e.range;
  var editedColumn = range.getColumn();
  
  if (editedColumn == 7) { // '계약명' 열이 수정되면
    // 계약명에 맞는 데이터 API로 요청
  } else if (editedColumn == 32) { // '상태' 열이 '확정'으로 바뀌면
    // 해당 행 데이터 서버로 전송
  }
}

UrlFetchApp 주요 옵션

내부 TMS API 서버와 안정적으로 통신하기 위한 핵심 옵션입니다.

const options = {
  'method': 'post',
  'contentType': 'application/json',
  'payload': JSON.stringify(data),
  'muteHttpExceptions': true,
  'validateHttpsCertificates': false,
};
  • muteHttpExceptions: true - API 에러(4xx, 5xx) 발생 시 스크립트가 중단되지 않고, 응답 코드를 받아 직접 오류를 처리할 수 있게 합니다. (예: 사용자에게 '오류' 상태 표시)
  • validateHttpsCertificates: false - 내부망의 사설 HTTPS 인증서 사용 시 유효성 검사를 건너뛰어 정상 통신을 가능하게 합니다.
    *주의: 공인된 인증서가 아닌 개발/내부망 환경에서만 사용*

핵심 Apps Script 서비스

  • SpreadsheetApp: 시트, 셀, 범위(Range)에 접근하고 값을 읽거나 쓰는 등 Google Sheets를 제어하는 핵심 서비스
  • UrlFetchApp: 외부 시스템(TMS API 서버)과 HTTP 통신을 수행하여 데이터를 주고받는 서비스
  • PropertiesService: API URL 등 자주 변경될 수 있는 설정 값을 코드와 분리하여 안전하게 저장하고 관리하는 서비스
  • HtmlService / Ui: '로딩 중' 팝업이나 알림창 등 사용자 인터페이스를 생성하여 상호작용을 돕는 서비스

API 서버 요구사항

  • 엔드포인트: Apps Script의 요청을 받을 수 있는 외부 공개용 URL 엔드포인트가 필요 (POST 방식 지원)
  • CORS 설정: Google 도메인(`*.google.com`, `*.googleusercontent.com`)에서의 요청을 허용하도록 CORS(Cross-Origin Resource Sharing) 정책 설정 필요
  • 방화벽(Inbound): Google의 공식 IP 대역 전체에 대한 인바운드 트래픽을 허용해야 `UrlFetchApp`의 접근이 가능함
  • 데이터 형식: 요청과 응답 본문(body)은 JSON 형식을 표준으로 사용
